I want to upgrade the processor in my Premium SE. What is the fastest processor I can install?

AST designed the Premium SE around its proprietary Cupid architecture. Cupid technology placed the components most likely to require upgrading, system memory and system processor, on a removable processor board. Components that typically remain constant resided on a separate system board. The user could upgrade the performance by simply replacing a plug-in processor board or adding SIMMs to a memory expansion board.

AST introduced Cupid architecture well over ten years ago, which is an eternity in the computer industry. In 1987, a 486 system was street priced at $10,000, so a $3,000 upgrade was a viable alternative. Unfortunately, the cost of reproducing that technology remains the same, while the price of new technology is dropping drastically. These Cupid upgrade prices have not been well received by our customers. In the customer’s own words, “I could buy a whole new system for that price!”.

AST realizes that the feasibility of upgrading to the higher performance processors with Cupid architecture is not competitive in today’s marketplace. After thoroughly evaluating both AST and non-AST solutions to provide price competitive alternatives to Cupid upgrades, AST has decided not to invest additional engineering resources researching Cupid board redesign and third party compatible alternatives. In essence, AST will no longer offer Cupid processor and memory board upgrades. AST regrets the inconvenience that this decision may cause our users with Cupid systems.

I upgraded the memory in my Manhattan server. Why doesn’t the operating system recognize the new memory?

If you have an EISA-based system and there is an EISA card installed or you have run the EISA Configuration Utility, you need to run the EISA Configuration Utility again to configure the new memory. The amount of memory shown in the EISA Configuration Utility should be modified to reflect the amount of physical memory in the system. The amount of memory can be set higher than the actual physical memory; it just can’t be less than then the amount.

Consider the following steps for updating the memory in the EISA Configuration Utility:

  1. Boot using the EISA configuration diskette and choose menu options Configure Computer, View and Edit Details.
  2. Press <F6> in order to edit the amount of memory installed.
  3. Press <10> after you enter the desired amount of memory,
  4. Save and Exit the configuration utility.
  5. Set the memory location of the NCR chipset well above physical memory. To do so, follow this procedure:
  6. Locate the View and Edit Details section. Scroll to the bottom of the screen marked Embedded PCI Controller.
  7. Highlight PCI Function and press <F6>.
  8. Change the memory address range to at least 20 megabyte higher than the total physical memory installed in the server.
  9. Change the memory address for all Embedded PCI controllers listed.
  10. Press <F10> to save and exit the configuration utility.

JEDEC/ parity (36-bit)/ Fast Page Mode/ 70ns/

I would like to install another processor in my Manhattan Server for a dual-processor configuration. Can I upgrade both processors to a faster speed?

The secondary processor socket in your Manhattan system is designed for dual processor or processor upgrade configurations. For multi-processing applications, the secondary chip must match the speed of the factory-installed processor. For example, you may not install a 200MHz secondary processor in a 133MHz system. Therefore, if the system shipped with a 133MHz processor, the second processor must also be 133MHz.

I am having problems with my Manhattan Server. What Diagnostics Utility should I use?

Using the Server Advanced Diagnostics can help you to identify problems with your system's hardware. For Manhattan SMP systems, use Advanced Diagnostics for SMP systems version 1.12. You can download the following file:

SMP3.EXE Advanced Diags For SMP v1.12

For all other systems, use Server Advanced Diagnostics version 2.23. Download the following file:

10450316.EXE Server Advanced Diagnotics v2.23

For accurate results, remove all add-in cards before you run the diagnostics.

How do I access System (CMOS) Setup on my server?

On most AST systems, you should see a message during a cold boot that prompts you for a keystroke or a series of keystrokes allowing you to access the System Setup. On some AST systems, this feature is available on warm and cold boots. You can refer to the following table as a reference guide:

Manhattan G <Ctrl+Alt+Esc>
Premium SE <Ctrl+Alt+Esc>
Premmia SE <Ctrl+Alt+Esc>
Manhattan P <F2>
Manhattan V <F2>
Manhattan S <Delete>
Manhattan Commerce Pro <Delete>
Manhattan D <F1>
Manhattan Presence Pro <F1>
