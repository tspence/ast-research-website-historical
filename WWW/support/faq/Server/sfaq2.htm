How do I Configure a Cupid Memory Expansion Board 202387-005?

The memory expansion board uses 1MB or 4MB, 80ns or 70ns write per bit SIMMs and can be installed in single SIMM increments. 1MB SIMMs must be installed before 4MB SIMMs. A maximum of 32MB (4MB SIMMs x 8 SIMM sockets) can be installed. Up to two CUPID memory boards can be installed. SW2-3 must be turned ON when used with the 202465-XXX processor board. The total system memory is dependent on the processor and processor board.

IMPORTANT NOTE:

In order for the memory board to work under most operating systems, including Novell NetWare and Windows NT, you must run the EISA configuration utility. Once in the configuration, select "Configure Computer" then "View or Edit Details." Scroll down to the memory board in the list, highlight it, and press F6 to edit the resources for the card. Set the starting address for the memory board to the top of previously configured memory. Also, set the memory amount to the amount of memory installed on the memory board only. Example: If the processor board is configured with 16MB and the memory board with 32MB, Starting Address = 16 and Memory Amount = 32.

I upgraded the memory in my Manhattan server. Why doesn’t the operating system recognize the new memory?

If you have an EISA-based system and there is an EISA card installed or you have run the EISA Configuration Utility, you need to run the EISA Configuration Utility again to configure the new memory. The amount of memory shown in the EISA Configuration Utility should be modified to reflect the amount of physical memory in the system. The amount of memory can be set higher than the actual physical memory; it just can’t be less than then the amount.

Consider the following steps for updating the memory in the EISA Configuration Utility:

  1. Boot using the EISA configuration diskette and choose menu options Configure Computer, View and Edit Details.
  2. Press <F6> in order to edit the amount of memory installed.
  3. Press <10> after you enter the desired amount of memory,
  4. Save and Exit the configuration utility.

Set the memory location of the NCR chipset well above physical memory. To do so, follow this procedure:

  1. Locate the View and Edit Details section. Scroll to the bottom of the screen marked Embedded PCI Controller.
  2. Highlight PCI Function and press <F6>.
  3. Change the memory address range to at least 20 megabyte higher than the total physical memory installed in the server.
  4. Change the memory address for all Embedded PCI controllers listed.
  5. Press <F10> to save and exit the configuration utility.

Why does Windows NT™ 3.51 fail to install on a Manhattan P / V system?

When you install Windows NT 3.51 from a CD-ROM/floppy disk package, the Windows NT 3.51 installation and setup programs may fail on the Manhattan P/V servers. The failure may exhibit itself in three ways:

  • After the setup program has copied the three (3) setup disks and the system performs its first re-boot to load the operating system, the system will hang at the Windows NT blue screen. You will see a fatal error message warning indicating that a hard disk error has occurred.
  • While copying files from the third setup floppy, a message will indicate that a number of files could not be copied. The files will have *.DLL extensions. If you continue the installation by pressing the IGNORE button at each file-copy error, the installation will indicate that it has completed successfully. When the system reboots, an error indicating that GINA.DLL could not be found, and the Windows NT blue screen will appear with a fatal error.
  • During the setup portion of installation, when Windows NT 3.51 attempts to begin copying files to the hard disk, the system may issue an error indicating that a hard disk could not be accessed. The system will instruct you that the installation has not been successful.

A temporary workaround is available, and is documented below. The workaround installs Windows NT 3.51 from the CD-ROM only. A floppyless installation may be done from the standard Windows NT 3.51 CD-ROM.

You can follow this procedure to complete a floppyless installation of Windows NT 3.51:

  1. Boot from a DOS boot diskette, loading the NCR SCSI drivers in the CONFIG.SYS file of the boot floppy. The NCR drivers are essential to access the CD-ROM drive attached to the NCR controller.

    For the Manhattan BIOS version 1.13 and older, load the NCR drivers (p/n 911283-313) for CD-ROM support in the following order:
    DOSCAM.SYS
    SCSIDISK.SYS
    CDROM.SYS /D:name
    ASPICAM.SYS

    name=name of CD-ROM drive (ex: drive1)

    These drivers are contained in the DOS subdirectory of the NCR PCI SCSI device drivers diskette. If you do not have the NCR driver diskette, download 283313.EXE Manhattan P/V/G NCR SCSI Drivers V.1.06

    For Manhattan BIOS version 1.14 and later, obtain the new drivers from Symbios Logic for the 4.0 SDMS PCI SCSI chipset. If you fail to use the updated drivers for BIOS version 1.14, you may corrupt or lose data. You can download these drivers from the Symbios Logic Web site:

  2. Load the NCR drivers for CD-ROM support in the following order:
    ASPI8XX.SYS
    SYMDISK.SYS
    SYMCD.SYS /D:name
    name=name of CD-ROM drive (ex: drive1)
  3. Add the following DOS file to the AUTOEXEC.BAT on the boot disk:
    MSCDEX /D:name  (name=name of CD-ROM drive. Name must match name given in CONFIG.SYS file entry for CDROM.SYS or SYMCD.SYS).
  4. Make sure that the boot disk contains the following DOS files:
    FDISK.EXE
    FORMAT.EXE
  5. Create a partition on the hard disk using FDISK. We recommend that the boot partition for the first Windows NT drive be 500MB or less.
  6. Format the partition.
  7. Change to the CD-ROM drive and change directories to I386.
  8. Enter the following command:
    WINNT /X /B

This command will start a floppyless installation of Windows NT. The system should detect the DPT (if installed) and NCR controllers at the appropriate stage of the installation.

How do I Install the Latest BIOS Revision and the latest version of EISACU on the Manhattan P & V?

NOTE: You should follow these procedures precisely and in the proper order to prevent any future problems once the system BIOS has been updated.

Prepare the BIOS diskette and flash the BIOS:

  1. Prepare the BIOS 1.12 and EISACU 1.09 diskettes. Listed below are the BIOS updates you can download:
    05547311.EXE Manhattan P/V Flash BIOS Update, v1.13
    11371040.EXE EISACU For Manhattan P/V,v1.09 644076
    Each self-extracting file will contain a batch file called EXTRACT.BAT that will create a single 3.5" diskette.
  2. Access System (CMOS) Setup by pressing <F2> at boot and press <ESC> once you are in the main menu.
  3. Choose "SAVE CHANGES, EXIT, and UPDATE BIOS."

After you see several on-screen instructions, the system will reboot causing the screen to go blank for a short time. The system will start a program that automatically updates the system's BIOS. Once the update is finished, you will see a prompt to restart the system.

Clear the EISA CMOS configuration:

  1. Verify that the system did update to the appropriate BIOS revision.
  2. Turn of the system and locate SW1 on the system board. You can find this switch block on the system board below the power supply and floppy drive connectors.
  3. Turn switch 4 to the ON position, and then turn on the system.
  4. Let the system complete POST, and then turn off the system.
  5. Return switch 4 to the OFF position and turn on the system. You should receive an "NVRAM bad" error.
  6. Enter System (CMOS) Setup by pressing < F2>, and then press <ESC> once you arrive at the Main Menu. Choose the option to "SAVE and EXIT."
  7. Boot to the EISACU 1.09 diskette and follow the proper procedures for configuring the EISA and PCI portion of the system. If prompted whether or not you wish to use a backup *.SCI file, choose, "Do Not Use Backup *.SCI File."

How do I install ArcServe 5.01g on the Manhattan P, V, and G Series?

Follow the ArcServe installation procedures as documented by Cheyenne. When prompted for the tape driver that you want to load, choose "Adaptec ASPI SCSI Manager" from the list. Do not specify any of the optional parameters that the installation provides.

  1. Make sure that NCRSDMS?.DSK is loaded first in your STARTUP.NCF. "where ?" denotes the version of NetWare you are using (e.g., NCRSDMS3.DSK, NCRSDMS4.DSK). The DSK drivers should be dated 09/24/94 or later.
  2. Load ASPICAM?.NLM after your DSK driver. "where the ?" denotes the version of NetWare you are using (e.g., ASPICAM3.NLM, ASPICAM4.NLM). Do not load any Adaptec SCSI drivers and do not load TAPEDAI.DSK. They are unnecessary for proper usage of ArcServe on the AST Manhattan Series.
  3. Edit the ASTART.NCF file created by the ArcServe installation program and:

    After ARCSERVE.NLM, insert a 10 second delay
    After ASPI50.NLM, insert a 20 second delay
    After TAPESVR.NLM, insert a 30 second delay
    The following is a sample ASTART.NCF from a Manhattan server:
    Search Add SYS:\ARCSERVE\NLM
    LOAD SYS:\SYSTEM\STREAMS
    LOAD SYS:\SYSTEM\CLIB
    LOAD SYS:\SYSTEM\TLI
    LOAD SYS:\SYSTEM\SPXS
    LOAD SYS:\SYSTEM\NWSNUT
    LOAD BTRIEVE -p-4090 -f-20 -h-60 -l-20 -t-15 -u-0 -s-30 -m-512
    LOAD BSPXCOM -d-8192 -s-15 -w-3
    LOAD SYS:\ARCSERVE\NLM\ASDB
    LOAD SYS:\ARCSERVE\NLM\FAXLIB
    LOAD SYS:\ARCSERVE\NLM\ARCSERVE
    LOAD DELAYCMD 10 "LOAD SYS:\ARCSERVE\NLM\ASPI50 HOST-0"
    LOAD DELAYCMD 20 "LOAD SYS:\ARCSERVE\NLM\TAPESVR"
    LOAD DELAYCMD 30 "LOAD SYS:\ARCSERVE\NLM\CLIENTS"

How do I Move the NCR Chipset Out of Physical Memory Address on the Manhattan P and V?

The procedure listed below may address the following problems:

  • SCO UNIX hangs on the text file upon bootup.
  • Intermittent abends in Novell NetWare.
  • Intermittent screen dumps in Windows NT Server 3.5.
  1. Boot the server to the AST EISA Configuration diskette. From the CONFIGURE COMPUTER option, choose menu options, and then choose VIEW AND EDIT DETAILS.
  2. Highlight PCI FUNCTION 1 ENABLED from the section marked EMBEDDED PCI CONTROLLER. Press <F6> and change the memory address range to at least one megabyte (1MB) above the total physical memory installed in the server. If you have a dual-channel Manhattan P, then you must move the second channel one megabyte (1MB) above the previous channel. For example:
    32MB RAM in server = 33MB address for first, 34MB address for second channel.
  3. Press <F10>, save, and exit the configuration utility once you have changed the addresses for all the necessary channels.

I downloaded a BIOS update; how do I use it?

Before you install a BIOS update, you should always make sure you downloaded the appropriate file. Always consult BIOS.TXT for the latest BIOS update. If you are unsure about a BIOS update, then contact a technical support representative for more information. Listed below is the procedure you should follow once you download the BIOS update:

Note: This procedure refers to the root directory of the hard drive as C:\> . If applicable, substitute a different drive letter.

  1. Create a TEMP directory on your hard drive.
  2. Copy the BIOS update file that you downloaded to this empty directory.
  3. Enter the following command at the DOS prompt:
    CD\TEMP
  4. Enter the name of the file that you downloaded at the C:\TEMP> prompt. When you enter this command you will extract the files contained within this compressed file.
  5. Insert a formatted 3.5" floppy diskette in your A: drive and enter the following command at the DOS prompt:
    EXTRACT

Entering this command will create the actual Flash BIOS update diskette.

Consider the following procedures for completing the flash BIOS update on the Manhattan G, Premium SE, and Premmia SE systems:

  1. Reboot the computer, and enter System (CMOS) Setup.
  2. Insert the BIOS diskette in the A: drive, and then select the "Update BIOS" option from the Main menu.
  3. Select "Save changes, exit, and update BIOS." Then, follow the on-screen instructions for completing the BIOS update. If the BIOS update succeeds, the computer will beep twice. After the beeps, you can remove the floppy disk and reboot the computer.

When updating the BIOS on all other Manhattan Systems, view the README.TXT file for BIOS update information.
