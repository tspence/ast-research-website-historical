The hard drive field is not present in System (CMOS) Setup on my EISA-based system. How do I access this field?

When you first run the EISA Configuration Utility, several fields are extracted from System Setup that are only accessible when you run the EISACU program. If the hard drive field is not present in your system’s CMOS Setup program, you will need to run EISACU to access that field.

I have added memory to my system; although it is counted during POST, it is not recognized by the operating system. What is the problem?

If you have an EISA-based system and there is an EISA card present or you have run the EISA Configuration Utility, you should run the EISA Configuration Utility to configure the new memory.

How do I access System (CMOS) Setup on my Premium 286 system?

AST used two different BIOS chips with Premium/286 systems. Early versions of this system shipped with a Phoenix BIOS, while the later systems shipped with an AST BIOS. Each of these different BIOS versions require different utility diskettes to access the CMOS Setup utility, "ASTSETUP." Premium/286 systems that use the Phoenix BIOS do not offer the setup routine within the ROM BIOS. You must access ASTSETUP from the utility diskette.

Change to the drive containing the diskette and enter ASTSETUP.

If your AST system has an older Phoenix BIOS, you can download DIAGS35.ARC. You will need to use the ARC600.EXE file to unzip the DIAGS35.ARC file. Both of these files can be downloaded from our BBS.

If your Premium/286 has an AST BIOS, then there are two methods to access ASTSETUP.

To access ASTSETUP from the Utility diskette, change to the drive containing the diskette and enter ASTSETUP. If you need a copy of the utility diskette, download the following file:

PR286.ZIP

The system will respond with a message that should prompt you to press either <F1> or <Ctrl+Alt+Esc> for Setup.

I downloaded a BIOS update; how do I use it?

Before you install a BIOS update, you should always make sure you downloaded the appropriate file. Always consult the flash BIOS tables for the latest BIOS update. If you are unsure about a BIOS update, then contact a technical support representative for more information. Listed below is a procedure you should follow once you download a BIOS update:

Note: This procedure refers to the root directory of the hard drive as C:\> . If applicable, substitute a different drive letter.

Note: Not all BIOS updates follow this exact procedure. Always check the README.TXT file contained within the compressed BIOS file for any information about the BIOS update.

  1. Create a TEMP directory on your hard drive.
  2. Copy the BIOS update file that you downloaded to this empty directory.
  3. Enter the following command at the DOS prompt:
    CD\TEMP
  4. Enter the name of the file that you downloaded at the C:\TEMP> prompt. When you enter this command you will extract the files contained within this compressed file.
  5. Enter "EXTRACT" or double click on the EXTRACT icon. You will see instructions to place a formatted 3.5" floppy diskette in your A: drive.
  6. Follow the on-screen procedures to extract the flash BIOS update to the diskette in drive A:.
  7. Remove the new BIOS update diskette, reboot the computer, and enter System (CMOS) Setup.
  8. Insert the BIOS diskette in the A: drive, and then select the "Update BIOS" from the Advanced Options menu.
  9. Select "Save changes, exit, and update BIOS." This option will reboot the computer and read the flash diskette in the A: drive. Following the beep codes, the BIOS will update and then the system should reboot, saving the changes.

How can I get more than 16 colors on my Premium II computer?

The system board in the Premium II shipped with 256KB of video RAM. This amount of video RAM is not sufficient to display Super VGA images; it will not support more than 16 colors. It is not possible to upgrade the amount of video RAM on the system board.

However, it is possible to install a third-party, ISA-based display adapter to attain the higher resolutions and greater number of colors. When you install the new card, you should disable the integrated video chipset. To disable the video chipset, move SW1-1 in the bank of eight switches to the OFF position.

If you have problems after you install the new card, the problem may be related to a conflict with the system's BIOS. Access System Setup by pressing <Ctrl+Alt+Esc> during a cold boot. In System Setup, press <PageDown> until you reach Setup Screen Two; then look for the option that reads "Shadow BIOS." If it is set to "Sys and Video," change it to read either "Sys" or "None." You may need to try both settings.

When I turn my system on there is no video and I hear two long beeps followed by three short beeps. What does this mean?

The two long beeps followed by three short beeps often indicates that the CMOS is corrupted. Often this issue occurs with systems that have a flash BIOS. To resolve the issue, reflash the system’s BIOS. If you require assistance with this procedure, or are uncertain which file to use, check the flash BIOS tables or contact a technical support representative for assistance.

My system issues three short beeps followed by five long beeps (possibly another series of short beeps). What is the problem?

This beep code generally indicates bad or incompatible memory in the system. If you recently added memory to the system, try removing it. Does the problem go away? What happens if you install the additional memory only? Does the system function normally?

Check your system’s memory specifications. Do you have the correct memory installed? Does your system use tin-lead or gold-lead RAM? It is typically not a good idea to mix gold-lead and tin-lead SIMMs (DIMMs). When you mix two different conductors together in the same computer you can cause galvanization to areas on the system board.

Also, check your system’s memory configuration tables. You can usually find this table with the system's User's Manual. Some system boards require that you install memory using specific configurations (e.g., install the largest SIMMs first, etc). If you are uncertain as to which type of memory your system requires, or cannot find the correct configuration table, contact a technical support representative for assistance.

Make sure the integrated ethernet controller is enabled and configured correctly in System Setup. The AMD Ethernet controller should be configured for IRQ 9, DMA 7, and I/O address 300. The AMD Ethernet controller can be configured in the Ethernet section of System Setup. Make sure no other installed devices are using the aforementioned settings.

I want to upgrade the processor in my Premmia GX sytstem. What options do I have?

The Premmia GX is designed to be upgraded with an identical second processor for use as a multi-processor system. For instance, if your Premmia GX system shipped with a 90MHz Pentium® processor, you can install an additional 90MHz Pentium processor. AST does not support processor upgrades to faster Pentium processors, with the following exception:

If the part number of your Premmia GX system begins with the numbers 501700-???, the 90MHz processor(s) may be replaced with a 100MHz Pentium processor. Jumper changes are required for this upgrade; refer to your user’s manual, or contact one of our support representatives if you need assistance or more detailed information.

Please note that in order for the second processor to be “active”, you must have an operating system that supports multi-processer capabilities.

Device Manager in Windows® 95 shows that there is some sort of conflict with the CMD IDE controller. What do I need to do to resolve this issue?

If the conflict is on the CMD controller itself, you will need to follow the procedure outlined below:

  1. Click on the Start button, select Programs, and MS-DOS Prompt to open up a DOS box.
  2. Change directories so that you are in the \WINDOWS\SYSTEM directory.
  3. Rename the PCI.VXD file.
  4. Copy the PCIIDE.VXD file from the Windows 95 CD-ROM into the \WINDOWS\SYSTEM directory.
  5. Rename the PCIIDE.VXD file to PCI.VXD.
  6. Close the DOS box.
  7. Access the Device Manager.
  8. Remove each and every controller from Device Manager by clicking on Properties, and then remove the check from the box labeled Original Configuration. Return to Device Manager, highlight the device, and click on the Remove button.
  9. Exit Windows 95, and then shut down your computer.
  10. Reboot the computer.

Windows 95 should properly recognize the primary IDE controller, and 32-bit disk access should be re-enabled.

I installed Windows® 95 on my Bravo MS/MS-T 6180/6200 system, and then Windows failed to recognize the CD-ROM drive. What is the problem?

The retail version of Windows® 95 did not ship with the driver for the Intel 82371SB PIIX3 IDE controller used in these systems. The following document on Intel’s Web site addresses the issue in depth:

http://developer.intel.com/design/motherbd/IDEINFUP.HTM

Intel also provides a link to a utility that will install the correct driver for the IDE controller. Alternatively, you can download MSHDC.INF, and use the following procedure to resolve this issue:

  1. Open the Device Manager, expand the Hard Drive Controllers entry, and remove all controllers from the list.
  2. Insert the diskette with MSHDC.INF in the floppy drive.
  3. Double-click on the My Computer icon.
  4. Double-click on the icon for the floppy drive.
  5. Right-click on MSHDC.INF and click on INSTALL.
  6. Shutdown the system, and then turn it off.
  7. Open the cover for the computer, and then clear the CMOS Setup by changing the jumper located at J6D1. You will need to remove the jumper that is set to pins 20-22, and reinstall it on pins 18-20. Leave the cover off.
  8. Turn on the computer. You will see an error message that indicates the CMOS has been cleared.
  9. Turn off the computer, and then reset jumper J6D1 back to pins 20-22. Return the cover to the system.

After you turn the system on, the new MSHDC.INF should correctly identify the controller. It may respond with a message that Windows 95 has detected new hardware and ask you to install a driver for it. Do not install the default driver; this is what you just removed. You need to inform Windows to install the driver from your floppy diskette, and then install the Intel 82371SB driver for the primary IDE controller. Reboot the system again; it should now detect the secondary controller and the CD-ROM drive. If Windows fails to recognize the drive, then you may need to repeat the procedure.

I received a 10.01 text-error message when I boot the computer. What is wrong?

The 10.01 error message indicates the bootable partition is not active. The Primary DOS Partition was not set as an active partition

Follow these steps to make the bootable partition on your system active:

  1. Boot the computer from a bootable DOS diskette.
  2. Enter FDISK at the DOS prompt .
  3. Choose "2. Set active partition."
  4. Choose "partition one," the Primary DOS Partition. FDISK will activate the partition and return to the FDISK menu options screen.
  5. Press <Esc> to exit FDISK.
  6. Turn off the computer, and take the bootable DOS diskette out of the drive. Reboot the computer and your hard drive should be recognized properly.

My AST desktop system does not recognize a second hard drive I added to the system. What is the problem?

If the system has two IDE controllers (primary and secondary), you will need to determine the best configuration for your needs. Typically, the optimal drive performance configuration will have both hard drives installed on the primary controller and the CD-ROM drive (if present) on the secondary controller.

Make sure that you configure the appropriate "Master" and "Slave" jumper settings on both drives. If either drive is jumpered to be a Single, or both are jumped as Master (or Slave), the system will not correctly recognize the drives. Most hard drive manufacturers silk screen a jumper setting diagram or include a sticker on the drive that will help you set the appropriate jumper setting.

Make sure the drives are properly identified in System Setup. Unless you have an older 286 or 386 AST desktop system, you should set the hard disk to "Auto." This setting will allow the BIOS to detect and configure most hard drives using Logical Block Addressing (LBA). Note: LBA does not use the same drive parameters as ECHS (Enhanced Cylinders Heads and Sectors). The numbers of cylinders, sectors and heads in System Setup will not match the ECHS parameters recommended by the drive’s manufacturer. That is not a problem; in fact, most drive manufacturers recommend using LBA if it is available.

Another issue you may encounter is drive incompatabilities. A drive from one manufacturer may not function with a drive from another manufacturer when attached to the same IDE controller. Some drives will work with only one configuration. If you use hard drives from different drive manufacturers, contact the manufacturers and find out if there are any known issues between the two drives you own. Listed below are some links to some common drive manufacturers:

Make sure that you cable the drives properly. You may want to keep an extra cable handy; often a faulty IDE cable can be mistaken for a bad hard drive. It is less expensive to purchase another IDE cable than it is to buy or replace a hard drive.

Be aware of any operating system limitations. MS-DOS and early versions of Windows® 95 use a FAT 16 file system. This file system will recognize up to a 2GB partition. If the drive you install is larger than 2GB, then you need to partition the drive accordingly. Depending on the file system (FAT or NTFS) that you use with the Windows NT™ Server and Windows NT™ Workstation operating systems, the size of the drive and partitions you can use will vary. For more information, check Article IDs: Q114841, Q119497, Q155615, Q153941 on the Microsoft Knowledge Base. For more information about FAT, HPFS, and NTFS, refer to article Q100108 on the Microsoft Knowledge Base.

Do not forget to format the drive! The majority of most drive manufacturers require that you format the new drive when you first install it in the system. Often the drive will not be accessible, or you will see an error message stating "The device is not functioning properly!" until you format the drive. Be sure that you specify the correct drive letter or you may format the original drive that came with the system.

I would like to add another floppy drive to my system; is there anything I should be aware of?

Make sure that the grey cable that connects the floppy drives to the system board is attached securely to the end connector on the floppy cable. If you have a cable reversed, the light on the floppy drive will often stay on continuously.

Floppy drives are "sensitive" to their edge connector "position" on the grey floppy cable; in other words the floppy drive on the end of the cable is usually the "A:\" floppy drive and the drive on the secondary connector will be the "B:\" floppy drive. Make sure that the drives are jumpered (or switched) correctly to identify their respective positions on the floppy cable. You will need to specify the drive attached to the end connector as the drive 1 position. Refer to the following graphic for more information:

Additionally, make sure that the drive is properly configured within System (CMOS) Setup. Make sure the CMOS settings that identify the floppy drives are correct for each drive and drive position. If you are using an add-in floppy controller, make sure that you change the Floppy Adapter to Add-in.

I am having problems installing a Token Ring card in an AST business desktop system. What do I need to do to make this card work?

If are installing a token ring card in a desktop that has an integrated audio chipset, then make sure the card uses an interrupt other than IRQ5. AST uses this IRQ setting for the audio chipset in all of our systems that have an integrated audio chipset. If it is essential to use this IRQ, then your option is to disable the integrated audio chipset.

Additionally, all token ring cards use the default I/O address, A20. In our desktop systems that ship with an ATI mach64 video chipset, you need to change this address to A24. This change will eliminate any conflicts with the I/O addresses used by the video chipset.

I have installed some 8MB SIMMs into my Bravo LC 486 system, but they only seem to be recognized as 4MB SIMMs, why?

At the time of manufacture, Bravo LC 486 systems were built to recognize single-sided memory. Since most 2MB, 8MB, and 32MB SIMMs are double-sided, the system may not recognize the upgrade.

In my Bravo MS/MS-T system, I am having problems installing an internal modem in Windows® 95. What do I need to do?

Windows 95 will auto-add an integrated COM port even though the COM ports are disabled within System (CMOS) Setup. This feature will often cause problems when you attempt to add an internal modem on COM1 or COM2.

To resolve this problem, disable one of the serial ports (preferably COM2) to free up the resources for the modem. Configure the modem to use COM2 and IRQ3. Next, disable COM2 within the Windows Device Manager to eliminate any conflicts. These conflicts may occur because of Windows 95 Plug-and-Play capabilities. After the BIOS hands over the responsibility of the system's configuration to the Plug-and-Play operating system, the serial ports are detected again even when they are disabled in System Setup.

Consider the following procedure to install the modem:

  1. Disable COM2 in System Setup. To make this change, press the <Ctrl+Alt+Esc> key combination after a cold boot to access System Setup. Under the Ports option, set Serial Port 2 to Disabled.
  2. Start Windows 95 and then access the Device Manager.
  3. Select COM2 under Ports.
  4. Click on Properties.
  5. Remove the check from the box labeled "Original Configuration."
  6. Click OK to return to Device Manager. Do not Remove COM2. It should have a red X next to it. Leave it just like this.
  7. Open Control Panel.
  8. Select Add New Hardware.
  9. Allow Windows to auto-detect new hardware. Windows should find the COM port.
  10. Select Add New Hardware again. This time, do not allow Windows to auto-detect new hardware.
  11. Choose your modem from the list.

Once Windows notifies the Registry that the modem has been added, you may need to reboot the computer to use the modem.

What is AST's stance on Cupid upgradability?

INTRODUCTION

AST's proprietary 32-bit interface, CUPID architecture, links plug-in processor boards and memory expansion boards with the computer's system board. CUPID stands for Completely Universal Processor Input/Output Design. Several AST products, including the Premium and CUPID lines of desktop and tower computers, utilize CUPID architecture.

BACKGROUND

Most of the personal computers designed around the same time contained interdependent components, making upgradability virtually impossible. With the development of new, improved components, the user would have to replace the entire system in order to take advantage of new technology.

AST designed CUPID architecture to allow customers to purchase the right level of computer performance for their current needs. In the future, the customer could increase processing power without having to endure the inconvenience of replacing the entire system. CUPID technology placed the components most likely to require upgrading on a removable processor board. Those components that typically remain constant resided on a separate system board. The user could upgrade the performance by simply replacing a plug-in processor board or adding SIMMs to a memory expansion board.

CUPID TODAY

AST introduced CUPID architecture well over six years ago, which is an extended period of time and technological advancement in the computer industry. In 1987, a 486 system was street priced at $10,000, so a $3,000 upgrade offered a viable alternative. Unfortunately, the cost of reproducing that technology remains the same, while the price of new technology is dropping drastically. These CUPID upgrade prices have not been well received by our customers. In the customer's own words, "I could buy a whole new system for that price!"

AST realizes that the feasibility of upgrading to the higher performance processors with CUPID architecture is NOT competitive in today's marketplace. After thoroughly evaluating both AST and non-AST solutions to provide price competitive alternatives to CUPID upgrades, AST has decided not to invest additional engineering resources researching CUPID board redesign and third party compatible alternatives. In essence, AST will no longer offer CUPID processor and memory board upgrades. AST regrets the inconvenience that this decision may cause our users with CUPID systems.
