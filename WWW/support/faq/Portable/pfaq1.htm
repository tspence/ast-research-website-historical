Dock ID

All Ascentia P Full docks come with an EEPROM with the same Electrical Identification programmed inside. If you use multiple docks with identical IDs on your notebook, then your notebook will use the same docked profile in Windows® 95, regardless of where you are docked. This is an advantage for using multiple docks that are configured the same. However, if you frequently dock the system to full docks that are configured differently and you want to maintain separate dock profiles, then you can program the Electrical ID of each dock to be different. This will result in Windows 95 creating separate dock profiles for each dock.

To change the electrical ID of your dock, download the following program:

FDAID.EXE

How do I use "non-printer" hardware on my Ascentia 900N's parallel port?

When some third-party devices are connected to the Ascentia 900N parallel port, the software does not properly recognize the interrupt on LPT1. For example, the following error message may appear on the display:

IRQ 7 could not be found on LPT1

Older versions of software do not properly recognize the VLSI implementation of the super I/O chipset.

For other software conflicts, contact the third-party software manufacturer for the latest drivers that support the VLSI implementation.

I'm trying to reinstall Windows on my Advantage! Explorer or Bravo NB and it hangs on the second or third diskette. What do I do?

When you run Windows SETUP, select Custom Setup. When SETUP displays your system's information, make sure VGA is in the Display Type field. The Windows installation should now complete successfully.

After you complete the Windows installation, you need to install the proper drivers, which you can find on your Notebook Utilities Disk. This disk is one of the disks you made when you received the "MAKE YOUR DISKS NOW!" message. If you do not have this disk, you can create it by downloading and running the following file:

BNB_VID.EXE Bravo NB/Adv! Explorer video drivers 2.6

Once you create the disk, you can follow this procedure to change your system's video drivers:

  1. Double-click on the Windows Setup icon under the Main Program Group.
  2. Click once on the Options selection on the menu bar.
  3. Hold down the mouse button and scroll down to the Change System Settings... selection.
  4. Release the mouse button to view the selection screen.
  5. Click on the down arrow to the right of the Display line.
  6. Release the mouse button tobring up a listing of video drivers installed on your system.
  7. Scroll to the bottom of the list and select "Other Display (Requires Disk from OEM)."
  8. Insert the drivers diskette and change the path to the specified drive.
  9. Choose one of the drivers that includes "*Chips 655XX." For example, "*Chips 655XX Linear."

After you load the drivers, restart Windows.

Why does my PC Card (PCMCIA) not work after I run MEMMAKER?

When you run MEMMAKER, it will try to configure your boot files for optimal use. However, it will often change the EMM386 line of your CONFIG.SYS and disable the memory range needed for PCMCIA use. AST systems require an exclusion of the entire D000 range. Consider the following syntax for excluding that range:

DEVICE=C:\DOS\EMM386.EXE NOEMS X=D000-DFFF

Some cards will not require you to exclude the entire range. Check with your PC card vendor if you need more memory.

Also, MEMMAKER often adds the HIGHSCAN switch to the EMM386 line. If you see this switch, remove it. Due to the added demands of power management in a portable system, AST recommends removing HIGHSCAN from the EMM386 line. In most cases, HIGHSCAN will try to force TSRs (terminate and stay resident programs) and other drivers into memory that are normally used by other notebook functions.

My Premium Exec / Advantage NB 386 notebook hangs after MS-DOS 6.2X has been installed. What should I do?

If the notebook hangs during the HIMEM.SYS extended memory text, follow this procedure:

  1. Turn on your system. As soon as you see the message, "Starting MS-DOS...," press <F5>.
    This should bypass your AUTOEXEC.BAT and CONFIG.SYS files.
  2. Enter the following command:
    PATH=C:\DOS
  3. To edit your CONFIG.SYS file, enter the following command:
    EDIT CONFIG.SYS
  4. Search the CONFIG.SYS file for the line that contains HIMEM.SYS. When you find the line, you will need to add the "/TESTMEM:OFF" switch. Your HIMEM.SYS line should be similar to the following example:
    DEVICE=C:\DOS\HIMEM.SYS /TESTMEM:OFF
    This switch disables the memory test, which allows your system to boot without hanging.
  5. Save the file, exit the MS-DOS Editor, and reboot your system.

Can I upgrade the internal modem for the Advantage! NB 486SX/33 to a 14.4Kbps or higher?

This notebook uses an 8250 UART chip. The 8250 UART limits the speed of the serial port to a maximum baud rate of 9600bps in Windows, and will not allow a 14.4Kbps or faster external modem to operate reliably at a speed faster than 9600bps. The fastest modem AST recommends is a 9600bps modem.

Is the processor upgradable on any of the AST notebooks?

The processors on all AST notebooks are soldered onto the system board of the computer. The processor cannot be upgraded or replaced.

Do I need to make any changes to my Ascentia 910N if I install Windows 95?

The BIOS for the Ascentia 910N does not support the Windows 95 Plug-and-Play and Advanced Power Management features. You need to disable the Advanced Power Management option in Windows 95.

The steps to disable Advanced Power Management in Windows 95 are listed below:

  1. Click Start, Settings, and Control Panel.
  2. Double-click the System icon.
  3. Click the Device Manager tab.
  4. Double-click the plus sign to expand the System Devices branch.
  5. Double-click the Advanced Power Management option.
  6. Remove the check next to the box that labeled, “Enable Advanced Power Management support.”
  7. Click OK.

Why doesn't my keyboard or mouse work in Windows for Workgroups?

The Ascentia 900N system keyboard may hang or fail to operate properly under Windows for Workgroups, version 3.11. This problem is caused by the way the keyboard controller circuit is implemented on some systems. Install a separate keyboard driver which ignores the extra interrupt signals generated by these keyboard controllers. Windows normally uses an internal "Virtual Keyboard Driver" or VKD.

To install the new VKD.386 driver:

  1. Download the following file:
    VKD386.EXE This self-extracting file will create the driver.
  2. Exit Windows.
  3. Copy the file VKD.386 to your WINDOWS\SYSTEM directory.
    Consider this example:
    COPY A:\VKD.386 C:\WINDOWS\SYSTEM
  4. Change to the Windows directory.
    CD \WINDOWS
  5. Using the MS-DOS EDIT command to edit your SYSTEM.INI file:
    EDIT C:\WINDOWS\SYSTEM.INI
  6. Search for the following line: KEYBOARD=*VKD. While in MS-DOS EDIT, press <F3>, and enter "*VKD" to find it. Change this line to read like the following example:
    KEYBOARD=VKD.386
  7. Save the SYSTEM.INI file. In MS-DOS EDIT, press <ALT>,<F>,<S>,<Alt>,<F>,<X>.
  8. Reboot your system and run Windows for Workgroups 3.11. The keyboard and mouse will now operate properly.

My Ascentia P reports the floppy drive as a 5.25 inch drive, rather than a 3.5 inch drive; what do I need to do?

To resolve this issue on the Ascentia P, perform the following steps:

  1. Access the CMOS System Setup by pressing <F2>.
  2. Access the security menu, and check “Diskette access.” If it is set to Administrator change it to User.
  3. Select Not Installed, at the “Disk A:” option.
  4. Save the changes and exit the CMOS Setup.
  5. Reboot the computer into the Safe Mode by pressing <F8>. When you see the message "Starting Windows 95.”
  6. Select option 6, “Safe mode command prompt only.”
  7. Enter the following command to access the A: drive:
    DIR A:
  8. Choose abort.
  9. Reboot the computer, and access the CMOS System Setup by pressing <F2>.
  10. Select 1.44 or 3.5 inch drive, at the Disk A: option.
  11. Save the changes and Exit.
  12. Reboot the computer, and the floppy drive should work properly.
