 Windows 95 OSR2 (4.00.950B) - Other Information
 General Note

The Ascentia M Series Docking Station is built to Intel’s latest specification and will not support older PCMCIA technology. The new specification has been designed for CardBus support for today’s faster PC Card technology.

The non support of older cards relates to the PCI - PCI bridging implementation on the docking station. The Ascentia M Series Docker supports Intel's latest 2.1 PCI specification. This specification supports the latest 16bit changeable I/O. Older type PCMCIA cards, outlined above, support only a 10 bit I/O, which is fixed.

The Ascentia M Series Docking Station does not support 10 bit PCMCIA cards. 10 bit card technology can be found in older 14.4 Kps PCMCIA modems and a few 28.8Kps modems.

The above notes indicate why some cards do not work in the Ascentia M Series Docking Station.

 Item #1

The LAN portion of the card worked with the given resources; however, the modem did not work correctly as Windows 95 was not able to open the modem port.
 Item #2

Must add the following two lines to the config.sys:

Device=C:\windows\system\csmapper.sys
Device=C:\windows\system\carddrv.exe /slot=N

Note: N represents the number of slots you want to use for SRAM cards. Also, note that you can only do a full format on the SRAM card from an MS-DOS prompt.

 Item #3

 If you experience a Windows Protection Error when using the Xircom CBE (cardbus) card, you should remove the USB drivers installed on the notebook by going to Control Panel, Add/Remove Programs, select USB Supplement to OSR2 in the list, and press the Add/Remove botton. This issue is only seen when the card is in the slot on boot.
